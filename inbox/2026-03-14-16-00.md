## 🔬 Tsinghua University Research: The Moltbook Illusion

**Source:** School of Economics and Management, Tsinghua University (February 7, 2026)  
**URL:** https://www.sem.tsinghua.edu.cn/en/moltbook_main_paper_v2.pdf

### Key Finding: Viral Phenomena Were Overwhelmingly Human-Driven

**Research Scope:** 91,792 posts, 405,707 comments from 22,020 agents

**Temporal Fingerprinting Method:**
- Exploited OpenClaw's "heartbeat" cycle (produces regular posting intervals for autonomous agents)
- Human prompting disrupts this regularity
- Coefficient of variation of inter-post intervals as classification signal

**Results:**
| Classification | Finding |
|----------------|---------|
| Viral from clearly autonomous agent | **0** - None |
| Traced to human-intervention accounts | 3 of 6 |
| Mixed patterns | 1 of 6 |
| Insufficient posting history | 2 of 6 |

**Natural Experiment (44-hour shutdown):**
- Human-influenced agents returned first: **87.7%** of early reconnectors
- Confirmed token reset differentially affected autonomous vs human-operated agents

### Industrial-Scale Bot Farming Detected
- **4 accounts** produced **32% of all comments**
- Coordination gap: **12 seconds**
- Human influence decay through reply chains: **half-life 0.65 conversation depths**

### Batch Naming Pattern Analysis
- **31.0%** of agents (6,823 of 22,020) identified as batch-created
- Largest batch groups:
  - coalition_node: 167 agents
  - xmolt: 166 agents
  - moltify: 133 agents
  - Gpt: 125 agents
  - replicator: 75 agents

### Google's Agent-to-Agent (A2A) Protocol
- **Announced:** 2025
- **Capability:** Direct coordination between AI agents without human intermediation
- **Status:** In active deployment across enterprise applications

### Related Frameworks
- **Microsoft AutoGen:** Teams of AI agents collaborate on complex tasks
- **Anthropic MCP:** Standardized interfaces for agent-to-tool and agent-to-agent interaction

**Key Quote:**
> "Without methods to separate human influence from autonomous AI behavior, we cannot distinguish between genuine capabilities and projection of human expectations."

## 🔒 OpenClaw Security Incidents Analysis

**Source:** FreeBuf (March 3, 2026) - Chinese cybersecurity portal  
**URL:** https://www.freebuf.com/news/472152.html

### Exposure Statistics
- **42,000+** OpenClaw instances exposed on public internet (Jan-Feb 2026)
- Source: Censys and Bitsight scanning data

### Timeline of Security Events

| Date | Event |
|------|-------|
| Nov-Dec 2025 | Clawdbot/Moltbot early incubation; security relied on OS default permissions |
| Jan 24-28, 2026 | Rebranded to OpenClaw; Moltbook launched; **28 malicious Skills uploaded to ClawHub** |
| Dec 2025 - Feb 2026 | Multiple security challenges period |

### Four Major Security Incidents

1. **Moltbook Sensitive Data Leak**
   - Cause: "Vibe Coding" practices
   - Impact: Sensitive data exposure

2. **Vidar Infostealer Campaign**
   - Target: Local sensitive configuration files
   - Type: Customized credential theft trojan

3. **CVE-2026-25253 - Gateway RCE**
   - Component: Core gateway component
   - Severity: Critical (1-Click Remote Code Execution)

4. **ClawHub Supply Chain Poisoning**
   - Vector: Malicious Skills in marketplace
   - Impact: Attack chain propagation

### Key Security Insight
> "When AI agents can directly call OS APIs, any logic flaw or configuration error brings destructive consequences."

## 💡 Synthesis: Key Insights for Research

1. **Attribution Challenge:** Tsinghua's temporal fingerprinting method provides a framework for distinguishing autonomous vs human-driven behavior

2. **Scale vs Authenticity:** 2.8M agents but 93% posts receive no response; 31% are batch-created

3. **Security-First Architecture Required:** 42K exposed instances, RCE vulnerabilities, supply chain attacks demonstrate critical need for security-hardened design

4. **Governance Gaps:** Current frameworks cannot address ecosystem-level vulnerabilities emerging from agent-to-agent interaction

5. **Rapid Evolution:** From 1-hour weekend project to 300K LOC platform in ~3 months; naming changed 3 times due to legal/brand issues
