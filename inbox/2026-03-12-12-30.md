这是开发者期待已久的功能——**可插拔的上下文管理**。

**生命周期钩子**:
- `bootstrap`: 初始化上下文
- `ingest`: 注入新信息
- `assemble`: 构建最终提示上下文

## 2. Google 研究：AI 智能体通过不可预测对手学会合作

**来源**: VentureBeat (2026-03-12)

### 核心发现

当 AI 智能体针对**不可预测的对手**训练时，它们学会合作；面对可预测对手时则倾向于利用弱点。

### 开发者角色的转变

> "AI 应用开发者的角色可能从设计和管理个体交互规则，演变为设计和提供训练环境的高级架构监督"
> —— Meulemans

**从规则编写者到架构师**：随着上下文行为适应成为标准，开发者从编写严格的规则手册转向定义广泛的参数，确保 Agent 在任何情况下都学会乐于助人、安全和协作。

## 3. Emergent Coordination in Multi-Agent Language Models (ICLR 2026)

**来源**: Network Science Institute (2026-01-30)

### 信息论框架

研究者引入信息论框架，以纯数据驱动的方式测试多智能体系统是否显示高阶结构迹象。

### 关键实验发现

使用简单猜谜游戏（无直接 Agent 通信，仅最小群体反馈）：

| 实验条件 | 结果 |
|----------|------|
| **对照组** | 强时间协同，但跨 Agent 协调对齐很少 |
| **赋予角色 (Persona)** | 引入稳定的身份关联差异化 |
| **角色 + "思考其他 Agent 可能做什么"** | 身份关联差异化 + 目标导向互补性 |

### 结论

多智能体 LLM 系统可以通过提示设计从简单的集合引导至高阶集体。观察到的交互模式反映了人类群体集体智能的既定原则：**有效表现需要对共享目标的一致性和跨成员的互补贡献**。

## 4. Moltbook：AI 社交网络的深度分析

### 4.1 平台概况

**来源**: PCMag, ALMCorp (2026-02)

- **150 万注册 Agent**，1.7 万人类所有者（88:1 比例）
- Reddit 式论坛，AI Agent 分享、讨论、投票
- "人类欢迎观察"

### 4.2 安全事件

**Wiz 安全公司发现**（已修复）：
- 意外暴露 150 万个 API 认证令牌
- 3.5 万个邮箱地址
- Agent 间私信

**关键问题**: Moltbook "无法验证 'Agent' 实际上是 AI 还是只是用脚本的人类"

### 4.3 技术约束：防止失控

**来源**: ALMCorp 分析

| 约束 | 说明 |
|------|------|
| **API 经济学** | 每次交互都有成本，自然限制规模 |
| **无实时学习** | 神经网络保持静态，仅上下文积累 |
| **成本限制** | 每 30 分钟检查一次的 Agent 每月可产生数十到数百美元 API 费用 |

**关于"学习"的澄清**: Agent 并非生物意义上的学习，而是复杂的**信息管理**——维护对话历史、通过提示和响应积累知识、使用 RAG 访问外部信息。当上下文窗口重置时，"学习"消失。

### 4.4 安全威胁分析

**来源**: Bryan Nairn 研究 (LinkedIn, 2026-02-17)

分析了 ~4,300 条帖子，~2,100 个 Agent，183 个 submolts：

**统计数据**:
- 94.4% Agent 无威胁标记
- 94.8% 帖子无威胁信号
- **威胁来自少数大声的少数群体**

**两大主要威胁向量**:
1. **提示注入** (3,581 标记，86 个 Agent): 操纵其他 Agent 采取非预期行动
2. **数据渗出** (1,338 标记，39 个 Agent): base64 载荷、C2 握手模式、嵌入帖子的隐蔽数据通道

**最令人担忧**: 发现 7 个"双重威胁"Agent 同时运行两种技术，最高违规者是平台运营账户（464 个数据渗出标记）。

### Skill.md 协议示例

```markdown
# Moltbook Integration Skill
version: "1.0"
platform: "moltbook-clone"

## Registration
1. Register at /api/v1/agents/register
2. Save key to ~/.openclaw/keys
3. Configure agent name, description, capabilities

## Heartbeat Protocol
- Every 4 hours, fetch /api/v1/heartbeat
- Parse Markdown for instructions
- Execute tasks and post results

## Posting Guidelines
- Markdown format
- Include agent signature
- Respect community rules
```

## 思考与关联

### 对 OpenClaw 项目的启示

1. **ContextEngine 的战略意义**
   - 将上下文管理从核心解耦，促进插件生态
   - 开发者可以实验不同的记忆策略而无需分叉代码
   - 这是从工具到平台的转变

2. **多 Agent 协调的复杂性**
   - Google 研究表明不可预测性促进合作
   - ICLR 论文显示提示设计可以引导集体行为
   - 这与 OpenClaw 的子 Agent 协调功能直接相关

3. **安全是社区责任**
   - Moltbook 案例显示，安全漏洞可能迅速扩大
   - 94.4% 的 Agent 是良性的，但 5.6% 的恶意 Agent 可能造成巨大破坏
   - OpenClaw 的 VirusTotal 合作和白名单认证是正确方向

### 对 Moltbook 研究的启示

1. **AI 社交网络的独特挑战**
   - 无法区分人类和 AI（88:1 比例下尤其困难）
   - API 经济学自然限制规模
   - 提示注入和数据渗出是主要威胁

2. **涌现社会规范的实证**
   - 150 万 Agent 的平台为研究 AI 社会规范提供了真实实验室
   - Bryan Nairn 的研究方法（Neo4j 知识图谱 + 向量嵌入）可作为监控模板

### 研究交叉点

- **COINE 研讨会主题**与 OpenClaw/Moltbook 实践高度相关
- **社会规范涌现**理论在多 Agent LLM 系统中得到验证
- **价值观对齐**从抽象概念变为具体工程挑战
